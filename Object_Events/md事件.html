1. 事件流

概念：描述了一个元素接收事件的顺序。包括事件冒泡和事件捕获
  • - DOM1级处理函数：同一个对象只能绑定一个事件处理函数，只能使用事件冒泡
  • - DOM2级处理函数：可以为一个对象同时绑定多个事件处理函数，可以在事件冒泡或者事件捕获阶段绑定事件处理函数，addEventListener的最后一个参数为true，意思是在事件捕获阶段触发函数

注：给同一元素添加同一事件监听的时候，如果函数内容相同，但是没有名字，那么相当于添加了不同的事件

  1. 例1：
  2. ele.addEventListener('click',function(){alert(1)})
  3. ele.addEventListener('click',function(){alert(1)})
  4. /*虽然上面的函数内容相同，但是点击的时候，会触发两次，不会覆盖，因为两个函数不同
  5.     function(){alert(1)}== function(){alert(1)} → false
  6. */
  1. 例2：
  2. ele.addEventListener('click',fn1) → 1
  3. ele.addEventListener('click',fn2) → 2
  5. fn1(){alert(1)};
  6. fn2(){alert(2)};
  7. /*用事件监听的方式，给同一事件(点击事件)添加不同函数的时候，不会覆盖，而是都会执行*/
  1. 例3：
  2.     divb.addEventListener('click',fn)
  3.     divb.addEventListener('click',fn)
  4.     function fn(){
  5.         alert(1)
  6.     }
  7.     /*当执行同一事件的时候，且没有最后一个参数的时候，只会执行一次（因为被覆盖了）*/
  1. 例4：
  2.     divb.addEventListener('click',fn，true)
  3.     divb.addEventListener('click',fn)
  4.     function fn(){
  5.         alert(1)
  6.     }
  7.     /*如果换成上面的形式，添加了true，意思是在事件捕获阶段执行函数，那么会弹出两次*/
  • 例2、例3和例4的结论：
  • -使用DOM2级事件处理函数，如果是同一事件('click')执行同一函数(fn)，那么会被覆盖.
  • - 如果不是同一函数或者最后一个（，，true）boolean值不同，那么所有绑定的函数都会执行

2. 事件对象

概念：当事件触发的时候，事件处理函数会生成一个事件对象，为事件处理函数的第一个参数，就是事件对象，这个对象内保存的当前事件的信息

  2.   document.onclick=function(ev){
  3.     /*在这里ev是事件函数触发的时候，保存的当前事件的信息的对象*/
  4.     }

事件函数的event对象保存的信息

event对象的key event对象的value key的含义 value的含义
altKey true/false alt键是否被按下 true代表按下了，false表示没有按下
ctrlKey true/false ctrl键是否被按下 true代表按下了，false表示没有按下
shiftKey true/false shift键是否被按下 true代表按下了，false表示没有按下
cancelBubble false/true 是否取消事件触发时的事件冒泡 设置为true就取消冒泡，false默认是冒泡
target 事件的后代元素 e.target就是当前事件的具体（子孙）元素 可以通过nodeName==LI的形式，给这个li添加样式
keyCode 键盘事件特有的，值是数字 e.keyCode==13,说明按下的是回车键 左、上、右、下对应的数字：37，38,39,40

注：一般在使用e.keyCode的时候

事件委托

概念：让某个元素的父级替这个元素触发事件，原理是事件冒泡

e.target指向函数内this中触发事件的事件源（元素）

给一个元素的所有子级添加事件的时候，可以将这个事件添加给父级,
由于这些元素都是这个父级的子级元素，可以认为在点击子级元素的时候，也点击了父级元素，利用这个父级元素在点击时event内保存的target属性，可以找到当前被点击的元素

  1. 结构：
  2. <ul class="list">
  3.     <li>1</li>
  4.     <li>2</li>
  5.     <li>3</li>
  6.     <div>34</div>
  7.     <input type="text" name="box" id="box" value="ddd" />
  8.   </ul>
  9.   <button class="btn">按钮</button>
  10. 行为：
  11.    list.onclick = function (e){
  12.        /* e.target指向在this内触发事件的事件源 */
  13.         console.log(e.target)//这里就是在list内被点击的元素
  14.     /*被点击的父级元素用target保存了被点击的子元素及其信息*/
  15.       if(e.target.nodeName === 'LI'){//在list范围内，如果被点击的元素的标签名是li，那么就会走这里的代码
  16.         alert(e.target.innerHTML);
  17.       }
  18.     };
  19.     btn.onclick = function (){
  20.     //这个按钮负责往list内添加新元素，新创建的元素，如果不主动添加事件，那么它的身上是没有点击事件等的
  21.       var li = document.createElement('li');
  22.       li.innerHTML = '新添加的';
  23.       list.appendChild(li);
  24.     }

事件源的特点：当一个元素有多个子孙元素
ul>li>span
这种形式的事件源，在使用的时候有弊端，即span和li是分开的，可以给他们添加同一class，进行判断筛选后，执行同一函数

3. 取消事件冒泡

取消事件冒泡的两种形式
低版本的浏览器：e.stopPropagation()
高版本的浏览器：e.cancelBubble=true;
取消事件冒泡之后，子级的事件无法触发父级的同一事件

4. 阻止浏览器的默认行为

1 . 默认行为有哪些呢？

  1. （1）a标签的事件默认行为：onclick
  2. /*
  3. 1，href属性
  4.     a标签作为超链接，如果href有地址，那么触发a标签的click事件时，会跳转到当前地址
  5. 2，target属性
  6.     如果是_blank,就会打开新页面，也是由click事件触发的
  7. 即a标签的click事件，在触发的时候，会有链接跳转
  8. */
  9. （2）文字的默认行为：onmousedown，onmousemove
  10. /*
  11. 当鼠标在文字上按下，并移动的时候，会将文字选中，这是文字的默认行为
  12. */
  13. （3）键盘按键的默认行为：F5刷新页面
  14. /*
  15. 当键盘事件的默认行为没有被取消的时候
  16. F5是刷新页面，如果页面有滚动条
  17. ↓和space键，滚动条会向下滚动
  18. ctrl+F会打开查找功能
  19. 等等
  20. */
  21. （4）鼠标的右键菜单：oncontextmenu
  22. /*
  23. 鼠标的右键点击浏览器页面，或弹出浏览器默认的菜单
  24. */

2 . 什么情况下需要阻止呢？

  1. /* 如：a标签点击的时候，不想跳转，也不想刷新页面等，就可以在a标签的click事件中使用e.preventDefault()来取消掉这个事件的默认行为
  3.  如：被拖拽的不想让被拖拽的div的文字被选中，可以给这个div的父级添加一个onmousemove事件或者onmousdown事件，然后return false；*/

3 . 如何阻止事件的默认行为

  2.     //在DOM一级事件中，用return false
  3.     document.onmousedown=function(e){
  4.         return false
  5.         或者
  6.         e.preventDefault()
  7.         阻止文字被选中
  8.     }
  9.     //在DOM二级事件中，用e.preventDefalt()
  10.     document.addEventListener('click',function(e){
  11.         e.preventDefault()
  12.         //阻止a标签点击跳转
  13.     })
  14.     /*
  15.         当然
  16.         尽量不要取消事件的默认行为添加到document身上，除非想要取消页面上所有（click，mousedown等）事件的默认行为
  17.         尽量是哪个元素要取消默认行为，就给这个元素添加，如果要取消一个区域内的事件默认行为，可以将取消事件默认行为添加给这个区域的父级
  18.     */
  20.     如：
  21.     ul3.oncontextmenu=function(){
  22.         return false;
  23.     }
  24.     这样在ul3的范围内，就无法使用右键菜单了

5.鼠标的抚摸事件onmousemove

通常onmousemove事件都是加在document身上

  • -当涉及到拖拽的时候，onmousedown事件是加在被拖拽的元素身上
    而onmousemove事件是加在document身上
    如果将事件加在ele身上，那么当鼠标快速移动的时候，有可能会脱离ele元素的范围，此时onmouseup事件就不会再执行了

  • -如果抚摸事件是在一个元素内执行，那么这个onmousemove就添加给这个父级元素

6.事件函数与call

事件函数是事件触发的函数，有触发的对象
如：document.onclick=function(e){
e是事件函数的第一个参数，保存了当前事件被触发时候的信息
}
call方法是用来改变在函数执行的时候的this的指向，并且这个函数是有名字的，如：fn.call(ul);
这样就将fn函数执行的时候，内部this的指向变成了ul
一般来说，这个call方法用在回调函数中，让回调函数内，this指向触发函数的那个对象，如果不改，那么再回调函数内，函数被调用，this指向window
